Jigsaw Gift Web Game

Jigsaw Gift supplies a platform for people to create a customized Jigsaw puzzle using their own images and send it as a gift to their friends by sending the URL or through social network sharing.

Users can upload a picture from their local machine or online photo galleries, or just by simply submitting an image URL. Then the image will be cut into jigsaw tiles. The user can input a message and send it to the receiver, just like the message on a post card in our daily life. After that, the user will get a URL of that jigsaw puzzle for him to copy it into clipboard, or share it directly on social network websites, including facebook, twitter, google plus and so on.

The receiver can access the specific jigsaw gift through the URL. After solving the puzzle, a post card of this image will be shown. It can be flipped and show the message from the sender. And the receiver can also create another jigsaw gift and send it back to the previous sender.

  • Upload an Image

  • Preview the Uploaded Image

  • Input Some Message for the Receiver

  • Share the Puzzle through Social Networks

Where Have I Been?

I'm currently working on a small project of a note taking iPhone application. I'm learning BackboneJS framework now. So I used this project as an exercise.

It allows user to take notes with their current location information, similar to a check-in mobile application like Path. The difference is that it stores data locally which gives users more privacy. So it's very suitable for users to write diaries or travel journals to record their personal life. When a user saves his notes, besides the note title and content, the application can also store the latitude and longitude values of the current location. It then generates a static Google map showing the location in the note.

This application can list all the notes which the user has taken in two different orders - alphabetical order and location distance order.

The Influential Factors of Traffic Fatality Rates

This information visualization web page helps viewers to investigate the correlation relationship between traffic fatality rate and alcohol consumption or vehicle quantity.

The heatmap demonstrates the difference of traffic fatality rates in state level with color coded. The line and bar chart below displays either alcohol consumption or vehicle quantity for each state by the bars, according to viewers’ selection. And traffic fatality rate in shown by a line in this chart. It allows viewer to visualize the comparison between the two variables of all the states to figure out whether there is a correlation. By scrolling on the timeline on the top of the page, the viewer can get the data of a specific year in both two graphs. The line chart on the right displays all detailed data of selected state overtime from 2004 to 2009.

This application also provides nice interaction elements such as a timeline scrollbar, a checkbox for sorting animation and a list for selecting a specific state, which make users understand the data easily.

Personal Identity

This is a personal identity project which includes the design of a logotype, a business card, two web mockups, a tri-fold brochure and a promotional water bottle.

Idea: My first name is Lei (蕾). In Chinese, Lei means a budding flower. So I designed the logo that looks like a flower with leaves and stalk. The left part of the logo, which looks like a leaf and a stalk, is the initials of my name - LZ. (In the first version, since my full name “Lei Zhang” is too long for a logotype, I decided to only use the initial of my first name in the logotype.)

Color: The three colors I chose for the follower are colors that may be seen on real flowers. They are bright and vivid, so that the logo looks positive and energetic. The two green colors are suitable for leaves and the stalk. The color harmony is compound.

Typography: The typography I chose for the logo is Myriad Pro. It’s sans serif so that it looks simple and clean. It’s not script and shows balanced weight on every letters, so it’s easy to read and looks relatively formal.

eRent Mobile

eRent Mobile is a iOS application for online rent management and payment solutions for Ann Arbor landlords and property management firms. It provides the platform where residents, property managers and technicians can communicate on property maintenance.

The residents can submit work orders if they need help. Then the manager assign the work order to a technician. Technicians can view the work orders that are assigned to them, and go to the specific unit and fix the fixture accordingly. Also, users can submit comments and pictures to provide more details about the fixture. This platform makes residents get problems solved conveniently, helps technicians work efficiently and allows managers to monitor and record their property easily.

It's based on a MySQL database and a RESTful web service in PHP MVC mode. Frameworks that were used include JQuery, PhoneGap, JQueryMobile and CodeIgniter. It uses AJAX to exchange data with the server through the RESTful web service. Also, Git was used as the version control tool.

Housing Coordinator Network Web Database

Housing Coordinators Network (HCN) is a non-profit organization which supplies housing information for vulnerable groups. This web database provides easy access, point and click/drop down menus, and low on-going maintenance which will in turn reduce the overdependence upon one individual and inform the users of the latest available housing at a more efficient rate than what HCN currently experiences.

Users of this website include landlords, housing coordinators and other people who search through HCN’s housing databases. The website allows all users to view the housing information, but only allows landlords and housing coordinators to add, edit and delete it. Only the administrator can manage user accounts.

Since many target users are old people, this web database also provides a easy password reminding function, which can send an email to the user about the password.

Academic Network of UMSI Professors

In recent two decades, the co-authored articles increased rapidly because “big science” becomes a research trend in the academia.

This study identified the collaboration relationship between UMSI faculties and researchers from other organizations. The data was downloaded from Web of Science database. It included totally 536 articles, and 142 of them were more than one author. In the analysis result, the visualizations show that the collaborations increase gradually.

There are two major factors that affect the co-author relationship – common research interest and geographical distance. The universities that have information schools tend to have stronger collaborative relationships with UMSI and universities tend to have stronger cooperation with the universities that are closer to them. Besides, some changes of objective conditions, such as economic environment and technology development have also significant influence on the trend of academic collaboration.
